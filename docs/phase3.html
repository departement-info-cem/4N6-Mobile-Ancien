Améliorations variées

Dans cette phase 3, nous allons apporter des améliorations à votre projet.

Git 2 points Un minimum de 4 commits de tailles comparables. Tu peux viser un commit par appel au serveur implanté.
Multilingue

2 points Vous devez traduire tous les textes de l'interface graphique existante. Vous devez avoir toutes les locales fr couvertes et l'application doit être en anglais par défaut.

Toutes les portions de l'application doivent être traduites incluant les messages de notifications, les messages d'erreur etc.

L'orthographe doit être correcte, les textes ne doivent pas contenir de fautes dans la version française.

Gestion des erreurs

5 points Le serveur envoie des erreurs dans plusieurs cas. Si l'utilisateur n'est pas authentifié, s'il l'est mais que sa requête est mauvaise (validation non respectée ...).

Vous devez programmer un affichage d'un message d'erreur clair pour tous les cas possibles dans toutes les activités de l'application.

Vous devez prévoir un message d'erreur dans le cas où le réseau n'est pas disponible (vous pouvez tester en passant l'appareil en mode avion).

Tous les messages d'erreur doivent être traduits pour les 2 langues.

Attente dans l'interface graphique

1 point Vous devez inclure un délai côté serveur de 2 secondes au début de chaque méthode avec un Mapping afin de permettre de voir le délai.

4 points Quand vous envoyez une requête au serveur, la réponse n'est pas immédiate. Pendant l'attente, vous devez indiquer à l'utilisateur qu'il se passe quelque chose et désactiver l'action qu'il vient de déclencher pour éviter qu'ils envoient des rafales de requêtes.

Pour chaque consultation (liste et consultation) vous devez indiquer à l'utilisateur que le chargement est en cours et afficher les informations quand elles sont reçues.

Pour toutes les actions de l'utilisateur, vous devez indiquer l'attente mais également désactiver l'action.

L'interface doit revenir dans un état fonctionnel après le chargement, que le résultat soit correct ou une erreur.

Sécurité / failles côté serveur

Il y a plusieurs failles ou défauts dans le serveur qui vous a été fourni pour le cours. Il peut s'agir de tous les types de problèmes qui ont été évoqués pendant le cours. Vous devez un problème et le documenter voire le régler

  • Vous devez créer un fichier SECURITE.md à la racine de votre projet Android
  • 2 points Dans une section FAILLE, expliquer quelle est la faille au niveau du code, l'erreur de programmation ou la faille de sécurité.
  • 2 points Dans une section EXPLOIT, expliquer précisément comment un attaquant peut tirer profit de la faille pour nuire au système ou à un utilisateur du système. Donnez les étapes détaillés afin qu'un autre étudiant puisse reproduire les étapes.
  • 1 point Dans une section CORRECTIF, expliquer comment combler la faille et éviter l'exploit. Vous pouvez indiquer quels fichiers de code modifier et donner le code source. Vous pouvez également pointer sur un commit d'un fork privé si vous en avez un.

Fonctionnement global

5 points négatifs Votre application doit fonctionner sans plantage et correctement. Ce pointage fonctionne en négatif. Si l'application fonctionne correctement en tout temps, vous conservez votre note. Dans le cas contraire, vous perdez des points avec un maximum de 5.

  • plantage de l'application -1 point
  • interface illisible -1 point
  • information affichée incohérente -1 point
  • autre cas ...
